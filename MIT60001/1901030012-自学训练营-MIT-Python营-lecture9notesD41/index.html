  1. Github Pulling changes from a remote repository

git pull is a convenient shortcut for completing both git fetch and git merge in the same command:

Because pull performs a merge on the retrieved changes, you should ensure that your local work is committed before running the pull command. If you run into a merge conflict you cannot resolve, or if you decide to quit the merge, you can use git merge -- abort to take the branch back to where it was in before you pulled.

  1. Syncing a fork

  • 2.1 Open Terminal.

  • 2.2 Change the current working directory to your local project.

  • 2.3 Fetch the branches and their respective commits from the upstream repository. Commits to master will be stored in a local branch, upstream/master.

  • 2.4 Check out your fork’s local master branch.

  • 2.5 Merge the changes from upstream/master into your local master branch. This brings your fork’s master branch into sync with the upstream repository, without losing your local changes.
